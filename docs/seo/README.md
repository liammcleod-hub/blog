# SEO

Read this file first when using `docs/seo/`.

## Purpose

This folder is the Bastelschachtel SEO planning and execution layer.

Use it for article prioritization, category support strategy, landing-page direction, metadata framing, and internal-link logic.

## What Lives Here

- SEO audits
- execution checklists
- category copy direction
- master-article planning
- page-level messaging and search framing

## How To Use This Folder

Start here when you need:
- the current SEO priorities
- article or landing-page direction
- category support strategy
- internal-link structure
- search-intent framing for Bastelschachtel categories

## Recommended Reading Order

1. This `README.md`
2. the most current master-article planning doc
3. the most current category-intro or category-support docs
4. the most current SEO audit or execution checklist
5. older SEO files only when you need additional context

## Working Rule

- Use the latest planning docs as the default source.
- Use audits and older decks as supporting context.
- Do not hard-code one dated file as the permanent SEO entrypoint; this folder will evolve.

## Typical Uses

- choosing the next article
- shaping a master article
- aligning blog posts to category pages
- rewriting collection intros
- prioritizing SEO fixes
