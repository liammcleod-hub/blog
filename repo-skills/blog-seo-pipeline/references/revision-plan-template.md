# Revision Plan Template

Use this structure after `qa-article` when the user wants changes applied, or as a bridge into `revise-article`.

## Objective

State the article goal in one sentence:

- topic
- format
- intended audience or angle

## Required Fixes

List the changes that must happen before the article is publishable.

Focus on:

- unsupported claims
- wrong or weak product links
- structure mismatch
- SEO gaps
- weak source handling

## Recommended Improvements

List quality improvements that are useful but not mandatory.

## Edit Strategy

State whether the right move is:

- surgical patch
- section rewrite
- full article rewrite

Default to the smallest change set that produces a defensible result.
