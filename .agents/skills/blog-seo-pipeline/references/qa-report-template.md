# QA Report Template

Use this structure when running `qa-article`.

## Findings

List issues in descending severity.

For each finding include:

- severity
- problem
- evidence
- consequence
- recommended fix

## Coverage Checks

Summarize pass/fail or strong/weak judgment for:

- dossier grounding
- brief alignment
- product-link integrity
- SEO coverage
- structure and format match
- source integrity
- information gain

## Priority Fixes

List the smallest set of changes that would materially improve publishability.

## Residual Risks

Call out anything that still cannot be verified because source artifacts or persistence are incomplete.
